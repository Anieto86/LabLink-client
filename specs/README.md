# Spec-Driven Development

This folder is the source of truth for feature behavior.

## Workflow
1. Write or update `spec.md` before coding.
2. Break the implementation into `tasks.md`.
3. Record decisions and tradeoffs in `decisions.md`.
4. Implement code in `src/features/<feature>/`.
5. Add or update tests in `src/test/<feature>/`.

## Done Criteria
- Spec acceptance criteria are satisfied.
- Implementation follows feature boundaries (`api`, `model`, `ui`).
- Tests cover the behaviors defined in the spec.
